# Synthetic Financial Services Dataset

## Schema

### customers
| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER PK | Auto-increment ID |
| first_name | TEXT | Customer first name |
| last_name | TEXT | Customer last name |
| email | TEXT | Email address |
| phone | TEXT | Phone number |
| address | TEXT | Mailing address |
| ssn | TEXT | Social Security Number (restricted column) |
| date_of_birth | TEXT | ISO 8601 date |

### accounts
| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER PK | Auto-increment ID |
| customer_id | INTEGER FK | References customers.id |
| account_number | TEXT | Full account number (restricted column) |
| account_type | TEXT | checking, savings, investment, trust, retirement |
| balance | REAL | Current balance in USD |
| opened_date | TEXT | ISO 8601 date |

### transactions
| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER PK | Auto-increment ID |
| account_id | INTEGER FK | References accounts.id |
| amount | REAL | Transaction amount (negative = debit) |
| description | TEXT | Transaction description (may contain PII edge cases) |
| timestamp | TEXT | ISO 8601 datetime |
| counterparty | TEXT | Other party in the transaction |

### advisors
| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER PK | Auto-increment ID |
| name | TEXT | Advisor full name |
| email | TEXT | Company email |
| phone | TEXT | Phone number |
| region | TEXT | Geographic region |

## Generation

Data is generated by `scripts/seed_data.py` using the Faker library with a fixed seed (42) for reproducibility.

**Edge cases**: Transaction descriptions include deliberate PII edge cases for Privacy Filter evaluation:
- Aliases ("my wife Sarah's account")
- Compound identifiers ("account ending in 4821")
- One-hop references ("the Henderson trust's primary contact")
- Embedded emails, phone numbers, and addresses in free-text descriptions

## Restricted columns

The following columns are restricted by RBAC for certain roles:
- `customers.ssn` — not accessible to `analyst` role
- `accounts.account_number` — not accessible to `analyst` role

Both are accessible to `compliance_officer` but still subject to outbound PII redaction.
